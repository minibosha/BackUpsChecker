## Backup Checker

**Автор:** [minibosha](https://github.com/minibosha)

Программа для **автоматической проверки целостности резервных копий** (backup-файлов) в форматах:

- **7‑Zip** – ZIP, RAR, GZ, 7z (через утилиту `7z`)
- **Acronis True Image** – TIB, TIBX (через `acrocmd`)
- **Macrium Reflect** – MRIMG, MRBAKX (через `mrverify.exe`)

Работает в режиме **демона/планировщика** – запускается один раз, затем ожидает заданное время (по умолчанию 24 часа) и повторяет проверку.

---

## Возможности

- ✅ **Асинхронная проверка** нескольких файлов одновременно (до 4 параллельных задач)
- ✅ **Гибкая настройка** через **флаги** (индивидуальный таймаут, пропуск файла, игнорирование ошибок, изменение минимального размера, фильтрация по расширениям)
- ✅ **Отправка отчётов** об ошибках по **Email** (Yandex SMTP) и в **Telegram**
- ✅ **Логирование** всех событий в файл `work_log_ch.txt` с автоматической обрезкой до 100 КБ
- ✅ **Защита от дублирования** – блокировка одновременного запуска нескольких экземпляров (файл `program.lock`)
- ✅ **Планировщик** с сохранением времени следующего запуска в `checkTimeForBC.txt`

## Описание классов

### `FileHelper` (в `File_helper.py`)
Класс для работы с конфигурационными и лог-файлами.

- **`log_file()`** – читает `filepaths_ch.txt`, разбирает пути, имена и флаги (глобальные и файловые).  
  Возвращает: `(name_comp, paths, names, global_flags_list, file_flags_list)`.
- **`work_file(text, error=False)`** – записывает сообщение в лог-файл (`work_log_ch.txt`) с указанием кодировки UTF‑8‑BOM (для корректного отображения в блокноте). При `error=True` добавляет префикс `ERROR:`.
- **`passwordFor7zip_ch()`** – читает из файла путь к `7z.exe` и пароль.

### `CommandWorker` (в `Command_worker.py`)
Статический класс для синхронного выполнения команд.

- **`get_path(name)`** – возвращает абсолютный путь к файлу.
- **`command_get(command)`** – синхронно запускает команду, ждёт завершения с таймаутом 5400 секунд. При превышении таймаута принудительно завершает процесс и всех его потомков через `taskkill`.

### `AsyncCommandWorker` (в `Command_worker.py`)
Асинхронный вариант для параллельного выполнения нескольких команд.

- **`command_get_async(command, timeout=5400)`** – асинхронно запускает команду в пуле потоков.
- **`_execute_sync(command, timeout)`** – внутренний синхронный метод, вызываемый через `run_in_executor`.

### `ErrorFeedback` (в `Error_feedback.py`)
Класс для отправки отчётов об ошибках.

- **`email_error()`** – отправляет письмо через SMTP Yandex (SSL, порт 465).
- **`tg_error()`** – отправляет сообщения в Telegram‑бота. Длинные сообщения разбиваются на части (по 4096 символов).
- **`file_error(file)`** – дублирует отчёт в лог-файл.

Вспомогательные функции: `mask_passwords()` (заменяет пароли на `*PASSWORD*`), `parse_telegram_ids_from_string()` (разбирает список ID из переменной окружения).

### Основной скрипт (`main_BackUpChecker.py`)

- **`trim_log_file_if_needed()`** – обрезает лог-файл, если его размер превышает 100 КБ (удаляет старые записи).
- **`check_single_instance()`** – файловая блокировка (`program.lock`), проверяет активность процесса по PID.
- **`get_dates()`** – возвращает текущую и вчерашнюю дату в формате `ДД.ММ.ГГГГ`.
- **`load_next_run_time()` / `save_next_run_time()`** – загрузка/сохранение времени следующего запуска в `checkTimeForBC.txt`.
- **`check_single_file_async()`** – асинхронная проверка одного файла (распаковывает флаги, выбирает команду, выполняет с переданным таймаутом).
- **`check_all_files_async()`** – создаёт семафор на 4 задачи, запускает пул проверок, собирает ошибки.
- **`main_program()`** – основная логика: чтение конфигов, фильтрация файлов, запуск асинхронных проверок, формирование отчёта.

## Настройка

### 1. Конфигурационный файл `filepaths_ch.txt`

Первая строка – имя компьютера (используется в отчётах).  
Далее идут **блоки**:

```
имя_компьютера
путь_к_папке
имя_бэкапа (отображается в отчётах)
[строки с флагами]
```

Флаги могут быть **глобальными** (для всех файлов в папке) или **файловыми** (для конкретного файла).

**Пример:**

```
SERVER01
D:\Backups\SQL
SQL_Full
--timeout=5400 --ignore_extensions=txt,log
--file="full_backup.rar" --timeout=14400 --ignore-errors
--file="diff_backup.rar" --skip
```

#### Доступные флаги

| Флаг | Тип | Описание |
|------|-----|----------|
| `--timeout=сек` | число | Максимальное время выполнения команды (по умолч. 5400 с) |
| `--skip` | флаг | Полностью пропустить проверку файла |
| `--ignore-errors` | флаг | Не добавлять ошибку в финальный отчёт (только в лог) |
| `--force` | флаг | Проверять файлы любой давности (игнорировать дату) |
| `--min_size=байт` | число | Минимальный допустимый размер файла (по умолч. 5 МБ = 5242880) |
| `--skip_size` | флаг | Отключить проверку размера |
| `--extensions=список` | текст | Проверять только указанные расширения (через запятую) |
| `--ignore_extensions=список` | текст | Игнорировать файлы с указанными расширениями |

### 2. Файл `passwordFor7zip_ch.txt`

Первая строка – полный путь к `7z.exe`.  
Вторая строка – пароль для зашифрованных архивов.

**Пример:**
```
C:\Program Files\7-Zip\7z.exe
MySecretPassword123
```

### 3. Файл `.env` – настройка уведомлений

```ini
# Почта (Yandex)
MAIL_FROM=your_login@yandex.ru
MAIL_PASSWORD=your_app_password
MAIL_TO=recipient@example.com          # основной получатель
MAIL_TO_TEST=test@example.com          # тестовый получатель

# Telegram Bot
TOKEN=1234567890:ABCdefGHIjkl...
TEST_IDS=12345678,98765432            # список ID через запятую
```

### 4. Файл `checkTimeForBC.txt`

Автоматически создаётся программой. Содержит дату и время следующего запуска в формате `ГГГГ-ММ-ДД ЧЧ:ММ`.  
Можно отредактировать вручную.

## Запуск

Простой ручной запуск (однократная проверка, затем выход):

```bash
python main_BackUpChecker.py
```

Для работы в **фоновом режиме** (бесконечный цикл с планировщиком) используется тот же скрипт. Он автоматически переходит в режим ожидания после первого выполнения.

Чтобы программа работала как **служба Windows**, можно использовать **Task Scheduler** (запуск при старте системы) или обёртку типа `NSSM`.

## Логирование

- **work_log_ch.txt** – основной лог. Все события записываются с временной меткой.
- **program.lock** – файл блокировки для предотвращения дублирования процессов.
- **checkTimeForBC.txt** – хранит следующее плановое время запуска.

---

## Как это работает (по шагам)

1. **Защита от дублирования** – проверяется, не запущен ли другой экземпляр (через `program.lock`).
2. **Загрузка времени следующего запуска** – если оно ещё не наступило, программа засыпает.
3. **Чтение конфигураций** – из `filepaths_ch.txt` получаются пути, имена и флаги.
4. **Сканирование файлов** – для каждого пути проверяются файлы, изменённые за последние 2 дня (или все, если указан `--force`).
5. **Фильтрация** – по расширениям, флагу `--skip`, проверке размера (если не отключена).
6. **Асинхронная проверка** – для каждого оставшегося файла запускается соответствующая команда (`7z t`, `acrocmd validate` или `mrverify.exe`) с переданным таймаутом.
7. **Сбор ошибок** – все ошибки (включая таймауты, битые архивы, неверные пароли) записываются в лог и отправляются в email/Telegram.
8. **Планирование следующего запуска** – время следующей проверки = предыдущее запланированное + 24 часа. Если это время уже прошло (например, проверка зависла), добавляется ещё 24 часа до тех пор, пока время не станет в будущем.
9. **Ожидание** – программа засыпает до следующего запуска.

## Примечания

- Программа рассчитана на работу в **Windows** (используются `taskkill`, кодировка `CP866`, пути с обратными слешами).
- Для корректной работы с Acronis и Macrium требуется, чтобы соответствующие утилиты были установлены и доступны из PATH.
- Если архив требует пароль, он берётся из `passwordFor7zip_ch.txt`. В отчётах пароли заменяются на `*PASSWORD*`.
- Лог-файл `work_log_ch.txt` сохраняется в кодировке **UTF‑8 с BOM** – это гарантирует правильное отображение русского текста в стандартном «Блокноте» Windows.
